# 공고 분석 실행 절차

나라장터 공고 대시보드에서 AI 분석(rfp-analyzer)을 연결하는 전체 흐름입니다.

---

## 전체 흐름 요약

```
[탭1] 관심 공고 다운로드
       ↓
[탭2] 분석 대기 목록에서 PDF 경로 확인 + 복사
       ↓
[Claude Code] rfp-analyzer 에이전트로 분석 실행 → result.md 저장
       ↓
[탭2] 분석 리포트 자동 표시
```

---

## 단계별 상세 안내

### 1단계 — 대시보드 실행

```
scripts/run_dashboard.bat 더블클릭
```

브라우저에서 http://127.0.0.1:5050 열기

---

### 2단계 — 탭1: 관심 공고 다운로드

1. **공고 목록(탭1)** 에서 분석하고 싶은 공고 행의 **[다운로드]** 버튼 클릭
2. 버튼이 "다운로드중..." → "분석대기"로 바뀌면 완료
3. 이 단계에서 첨부파일이 자동 다운로드되고 HWP는 PDF로 변환됩니다

> 주의: 한컴 한글이 PC에 설치되어 있어야 HWP → PDF 변환이 가능합니다.
> 미설치 시 다운로드는 완료되지만 변환은 건너뜁니다.

---

### 3단계 — 탭2: 분석 대기 목록에서 PDF 경로 확인

1. **분석 리포트(탭2)** 클릭
2. "분석 대기 공고" 섹션에서 해당 공고 카드 확인
3. **PDF 경로 옆 [복사] 버튼** 클릭 → 클립보드에 경로 복사됨

---

### 4단계 — Claude Code: rfp-analyzer로 분석 실행

1. Claude Code를 열고, 새 대화 시작
2. 아래 형식으로 분석 요청:

```
rfp-analyzer 에이전트를 사용해서 아래 PDF를 분석해 주세요.

PDF 경로:
C:\...\파일명.pdf
```

3. rfp-analyzer가 9개 섹션 분석 리포트를 생성하고
   **자동으로** `analysis/{stable_id}/result.md` 경로에 저장합니다
   - stable_id는 PDF 경로(`scraper/output/나라장터/{stable_id}/`)에서 자동 추출
   - 저장 완료 후 에이전트가 저장 경로를 응답에 명시

> **사용자가 별도로 stable_id를 지정하거나 저장 경로를 안내할 필요 없습니다.**
> PDF 경로만 주면 저장까지 자동으로 완료됩니다.

---

### 5단계 — 탭2: 분석 리포트 자동 표시

1. 대시보드 탭2를 새로고침(F5) 하거나 탭 전환 후 재진입
2. "분석 완료된 공고" 섹션에 해당 공고 카드가 나타납니다
3. 카드를 클릭하면 전체 리포트(9개 섹션)가 인라인으로 표시됩니다

---

## 분석 결과 파일 위치

```
D:\02.Dev_Project\webscrap\
└── analysis\
    └── {stable_id}\
        └── result.md   ← rfp-analyzer가 저장하는 분석 리포트
```

result.md 파일이 존재하면 대시보드가 자동으로 분석완료로 인식합니다.
별도 버튼 누름이나 설정 변경 없이 파일 저장만으로 반영됩니다.

---

## 자주 묻는 상황

| 상황 | 원인 | 조치 |
|---|---|---|
| 탭2에 공고가 안 보임 | 아직 다운로드 미완료 | 탭1에서 다운로드 먼저 실행 |
| PDF 경로가 비어 있음 | 첨부파일 없는 공고 또는 HWP 변환 실패 | 공고 원문 사이트에서 직접 다운로드 |
| 리포트 클릭해도 내용 없음 | result.md 없음 | 4단계(rfp-analyzer 실행) 미완료 |
| 다운로드 버튼이 "분석대기"로 고정 | 이미 완료된 상태 | 탭2에서 분석 실행 가능 |
